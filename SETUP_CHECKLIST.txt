# Quick Setup Checklist

## Step-by-Step Implementation Guide

### ✅ STEP 1: Database Migration (REQUIRED)
1. Open phpMyAdmin
2. Select your database
3. Go to SQL tab
4. Copy and paste the contents of `migrate_optimized_orders.sql`
5. Click "Go" to execute
6. Verify new columns added to orders table:
   - customer_id
   - processed_by
   - payment_status
   - updated_at

### ✅ STEP 2: Test Customer Ordering
1. Open browser and go to: `http://localhost/torj/customer_order.php`
2. Browse the menu (products should load automatically)
3. Add items to cart
4. Enter customer name and table number
5. Click "Place Order"
6. You should see success message with order ID

### ✅ STEP 3: Test Barista View
1. Login as barista
2. Go to barista.php
3. You should see the order you just placed
4. Update status from "pending" to "preparing" to "ready"

### ✅ STEP 4: Test Cashier Payment Processing
1. Login as cashier or admin
2. Go to orders.php
3. You should see the order with:
   - Status: "ready"
   - Payment: "unpaid" (orange badge)
   - "Process Payment" button
4. Click "Process Payment"
5. Confirm the action
6. Order should now show:
   - Status: "completed"
   - Payment: "paid" (green badge)
   - No more "Process Payment" button

### ✅ STEP 5: Test Walk-in Orders (Optional)
1. Login as cashier
2. Go to neworder.php
3. Take order manually for walk-in customer
4. This should work as before

## Troubleshooting

### Problem: Products not loading in customer_order.php
**Solution:** 
- Check if products table has data
- Verify api/get_products.php is accessible
- Check browser console for errors

### Problem: "Process Payment" button not working
**Solution:**
- Verify you're logged in as cashier or admin
- Check api/process_payment.php exists
- Check browser console for errors

### Problem: Database error when placing order
**Solution:**
- Make sure you ran the migration SQL
- Verify all new columns exist in orders table
- Check db.php connection settings

### Problem: Order not showing in barista view
**Solution:**
- Refresh the page
- Check if order was actually created in database
- Verify barista.php is querying orders correctly

## Quick Test Script

Run this in phpMyAdmin SQL tab to verify your setup:

```sql
-- Check if new columns exist
DESCRIBE orders;

-- Should show: customer_id, processed_by, payment_status, updated_at

-- Check recent orders
SELECT 
    id,
    customer_id,
    processed_by,
    customer_name,
    status,
    payment_status,
    created_at
FROM orders
ORDER BY created_at DESC
LIMIT 5;
```
